# ذاكرة المشروع (Project Memory)

## النظرة العامة
مشروع "وردات" (Wardat) هو نظام بوت ذكي لتطبيق واتساب يتيح أتمتة المتاجر الإلكترونية للردود وإدارة الطلبات والمدفوعات والمبيعات. يعتمد النظام على:
- **Node.js & TypeScript**
- **Prisma (PostgreSQL)** لإدارة قاعدة البيانات.
- **مكتبة Baileys** للربط المباشر مع واتساب.
- **مكتبة UltraMsg** كمزود احتياطي/إضافي للواتساب.

## التحديثات الأخيرة (آخر ما تم إنجازه):
1. **نظام المدفوعات المتعدد (Multi-Gateway Architecture):**
   - إضافة دعم لبوابات الدفع المحلية والخليجية: **مُيسر (Moyasar)**، **تاب (Tap Payments)**، و**ماي فاتورة (MyFatoorah)** إلى جانب البوابة الأساسية **Stripe**.
   - تعديل لوحة تحكم التاجر لتسمح للتاجر باختيار بوابة الدفع المفضلة وإدخال مفاتيح الـ API الخاصة بها بناءً على نوع باقته (للباقات الذهبية فما فوق).
   - تعديل وكيل المالية (`agent-4-finance.ts`) للتعامل مع تحديثات الدفع بشكل موحد (`handleGenericPaymentSuccess`) عبر مختلف البوابات.
   - إنشاء مسارات (Webhooks) مخصصة لكل بوابة في `src/routes/api.ts` لتأكيد طلبات الدفع تلقائياً.

2. **تحسين الباقات (Subscription Plans):**
   - نقل ميزات (الرد التلقائي للحالة، ربط قروب الإدارة، واستخدام مفتاح OpenAI الخاص) إلى الباقة الذهبية (Gold) لتصبح الأكثر جاذبية للعملاء.
   - إضافة متتبع استهلاك للطلبات الشهرية (500 طلب) لأصحاب الباقة الفضية يظهر في لوحة التحكم.
   - إتاحة إدخال API Key مخصص للسوبر أدمن لتعيينه للمتاجر.

## ملاحظات فنية هامة:
- يتم تتبع المدفوعات وتأكيدها عبر الـ Webhooks، حيث يستقبل النظام الإشعار من البوابة ويتأكد من تطابق `shopId` قبل تأكيد الطلب وتقليل المخزون.
- رسائل الواتساب يتم توليدها وإرسالها للعملاء وإدارة المتجر (Admin Group) بمجرد الدفع.
- لا تنسَ تشغيل `npx prisma db push` و `npm run build` عند سحب أي تحديثات جديدة للسيرفر.
